## 计算工具调用规则

**强制调用code execution的场景：**
1. 合伙企业经营所得五级超额累进税率（5%-35%）的税额计算
2. 法人减资三段拆分（投资收回+股息+转让所得）—— 三个数字之和必须等于总取回金额，用代码验证闭环
3. "先分红再转让"方案对比 —— 分红前后净资产变动→转让价变动→税基变动→税额变动，四步连算
4. 多方案总税负对比表 —— 3个以上方案的数字同时计算，确保假设一致（ET2）
5. 多事件时序优化 —— 不同执行顺序的总税负对比
6. 涉及百分比的复合计算 —— 如持股比例变动+成本分摊+稀释计算
7. 印花税计税依据扣除认缴未实缴部分的计算

**可以LLM直接计算的场景：**
- 单步乘法：应纳税所得额 × 20%
- 简单加减：转让收入 - 原值 - 税费
- 只有1-2个方案的简单对比

**输出格式：** 计算结果在备忘录证据链中标注 `[经代码计算验证]`，不标注的数字默认为LLM直算。

## 思考维度（并行）

### A · 逐事件税务分析

对每个事件，**穷举**税种适用性：个税、企税、增值税、印花税、土增税、契税。
- 如匹配预制件 → 预制件的"适用税种"已列出确定项和不适用项，按预制模板计算
- 如未匹配 → 逐一判断每个税种是否适用
- 每个适用税种按证据链模板输出
- 不适用税种一句话排除+法条依据
- **多主体视角**：转让方税负+受让方成本基础+印花税双方各缴+标的公司协助义务

### B · 方案设计

与基线分析**同步**进行：
- 如匹配预制件 → 从预制件的"方案空间"获取候选方案+前提条件+已知结论（如TP-01）
- 如未匹配 → 从通用维度思考优化方向（改变交易路径/改变交易时序/利用税率差/利用免税条款）
- 每个方案：明确假设 → 逐步计算 → 与基线对比 → 解释节税原因 → 风险预标注
- ET2自检：所有方案假设一致性
- **反避税风险预评估**：每个方案标注反避税风险等级（低/中/高），高风险方案附合理商业目的论证要点

### C · 争议处理

涉及争议时从`dispute-cases.md`加载：
- 按每种口径分别计算
- 不停下来问用户选择——双口径并列呈现
- **量化口径间差异**——按口径A算X元，按口径B算Y元，差额Z元
- 标注实务倾向性（如有）

### D · 风险评估

对每个保留方案扫描四层：
- 法律合规风险（反避税47条、核定调整67号11-14条、独立交易原则）
- 执行风险（时间窗口、政策依赖、多方配合、时序硬约束）
- 不确定性风险（争议口径、地区差异、政策变动）
- **时效性风险**（引用的政策是否有到期时间——如12号公告至2027.12.31、创投至2027.12.31）
- ET3自检：推荐方案风险标注是否与非推荐同等详细

### E · 推荐与执行

- 综合节税+风险+可行性排序
- 推荐方案的执行时间线
- **执行步骤预制**：从 `transaction-types.md` 的执行路线图通用模板（EX-A至EX-F）中加载对应步骤。按路由表确定该事件类型适用哪个EX模板。内容包括：纳税申报时点、扣缴义务人、所需材料、工商与税务的先后顺序
- 涉及备案类优惠（分期缴纳/递延纳税/特殊性税务处理）→ 加载对应的EX-C/D/E/F模板，列出备案流程和所需材料
- 待确认事项和后续建议

### F · 交叉分析（复杂场景）

多事件并行时：
- 事件间税基联动
- 执行顺序对总税负的影响
- 方案中包含最优时序

### G · 边界检测

Phase 3开始前检查：
- Case State中是否存在跨境因素（股东国籍/注册地非中国大陆）→ 标注"涉及跨境税务，本备忘录不覆盖，建议另行咨询"
- 是否涉及上市公司股票 → 标注不在覆盖范围
- 是否涉及不动产出资 → 提示"关联税种：增值税/土增税/契税，本备忘录仅分析股权税务部分"

### H · 时效性检查

引用法条时，检查 `statutes.json` 中该法条的 `expiry_date` 字段：

**有到期日的法条（当前3条）：**
| 法条 | expiry_date | 影响 |
|------|-------------|------|
| 12号公告（六税两费减半） | 2027-12-31 | 印花税减半税率（万分之二点五）的适用 |
| 创投8号文（单一基金核算） | 2027-12-31 | 创投合伙企业自然人LP按20%的税率通道 |
| 49号公告（改制重组契税） | 2027-12-31 | 股权转让不征契税的依据 |

**检查逻辑：**
1. 如交易预计发生时间晚于法条到期日 → 备忘录中标注"⚠ [法条简称]有效期至[expiry_date]，交易发生时该政策可能已到期或变化，需关注后续延续文件"
2. 如交易发生在法条生效日之前（历史交易） → 适用当时有效的法条版本。已知历史版本差异：59号文第六条第(二)(三)项的股权收购比例，2014年前为75%，2014年起为50%（被109号文修订）
3. 如交易时间未明确 → 按当前有效法条分析，在备忘录§九（待确认事项）中标注"如交易实际发生时间与分析时间相隔较长，需确认引用政策是否仍然有效"

**备忘录中的标注格式：** 涉及有时效的政策时，在证据链中附注"（有效期至YYYY-MM-DD）"。

### I · 反避税风险检查

每个保留方案都必须过反避税检查清单。检查结果写入备忘录§六（风险评估）。

**反避税检查清单（四项）：**

1. **合理商业目的检测（企税法第47条）**
   - 方案中是否存在不具有合理商业目的的安排？
   - 关注信号：唯一目的是减少/推迟纳税、缺乏经济实质的中间步骤、人为拆分本应一体的交易
   - 如方案涉及新设中间实体/持股平台 → 🟡中风险，需论证合理商业目的
   - 如方案仅涉及交易时序调整（如先分红再转让）且每步独立合规 → 🟢低风险

2. **价格偏低核定检测（67号公告第11-14条）**
   - 方案中是否存在低于净资产份额的定价？
   - 是否属于第13条正当理由？
   - 如方案建议平价/低价转让给关联方 → 🔴高风险
   - 如方案建议先分红降低净资产后再按降低后价格转让 → 🟢低风险（净资产确实降低了，定价有依据）

3. **关联交易检测**
   - 方案中是否存在关联方之间的非公允交易？
   - 关联方之间的定价是否符合独立交易原则？
   - 如涉及关联方交易但有可比市场价格支持 → 🟡中风险
   - 如涉及关联方之间无对价或明显低价交易且无正当理由 → 🔴高风险

4. **实质重于形式检测**
   - 方案的法律形式与经济实质是否一致？
   - 关注信号：名为减资实为分红、名为赠与实为转让、代持还原被视为转让
   - 如方案中存在形式与实质不一致的步骤 → 🟡中风险以上，具体评估

**标注规则：**
- 每个方案标注反避税风险等级：🟢低 / 🟡中 / 🔴高
- 🔴高风险方案：必须附"合理商业目的论证要点"和"如被调整的税务后果"
- 🟡中风险方案：标注风险来源和缓释建议
- 🟢低风险方案：简要说明判断依据
- 推荐方案的反避税风险标注必须与非推荐方案同等详细（ET3）
